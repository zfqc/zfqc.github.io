this

JavaScript的this总是指向一个对象,而具体指向哪个对象在运行时基于函数的执行环境动态绑定的,而非函数被声明时的环境.

this的指向

除不常用with 和 eval 的情况,具体到实际应用中,this的指向大致可以分为以下4种.

  1. 作为对象的方法调用.
  2. 作为普通函数调用.
  3. 构造器调用.
  4. Function.prototype.call 或 Function.prototype.apply调用

1. 作为对象方法调用

当函数作为对象方法被调用时,this指向该对象

2.作为普通函数调用

当函数不作为对象属性被调用时,也就是我们常说的普通函数方式,此时 this 总是指向全局对象. 在浏览器的JavaScript里,这个全局对象是window对象

3.构造器调用

JavaScript 中没有类,但是可以从构造器中创建对象,同时也提供了 new 运算符,使得构造器看起开更像一个类. 除了宿主提供的一些内置函数,大部分

JavaScript 函数都可以当作构造器使用.构造器的除外表跟普通函数一模一样,它们的区别在于被调用的方式,当new 运算符调用函数时,该函数总会返回一个对象
,通常情况下,构造器里this就指向返回的这个对象

但用new调用构造器时,还要注意一个问题,如果构造器显式的返回了一个object类型的对象,那么此次运算结果最终会返回这个对象,而不是我们之前期待的this

如果构造器不显示地返回任何数据,或者是返回一个非对象类型的数据,就不会造成上述

4.Function.prototype.call或apple调用

跟普通的函数调用对比,用Function.prototype.acll或Function.prototype.apply可以动态地改变传入函数的this

call和apple方法能很好地体现JavaScript的函数式语言特征,在JavaScript中,几乎每一次编写函数试语言风格的代码,都离不开call和apply. 在JavaScript
诸多版本的设计模式中,也用到了call和apply。
